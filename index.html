Use cases

Do your Android work without living inside app menus.

The product is not another chatbot. It is a phone-side operator for everyday jobs that normally require tapping, searching, typing, and checking multiple apps.

01

Website and form work

Open Chrome, inspect a page, fill fields from the info you gave, and ask you in Telegram when a required value is missing.

02

App errands

Check prices, compare food delivery apps, open ride apps, read order status, and keep you updated.

03

Messaging and notifications

Forward app notifications to Telegram and reply into supported apps from the same control thread.

04

Files and reading

Open downloads, read documents, summarize pages, create PDFs, and send generated files back through Telegram.

05

Routine phone operations

Schedule commands, run macros, open apps, use maps, search YouTube, manage settings, and repeat your last task.

How it works

The brain is screen-aware, not menu-scripted.

BharatClaw should navigate like a person: observe the UI, decide the next step, act, verify, and recover when the screen is different from the ideal path.

Plan

Turns your request into a task plan

The router chooses a skill when the job is known, or falls back to the general screen agent for open-ended app work.

Observe

Reads the current Android screen

Accessibility data and screen context tell the agent what fields, buttons, lists, and warnings are actually visible.

Act

Taps, types, scrolls, opens apps

It uses controlled Android actions and avoids sensitive steps like entering UPI PINs or payment confirmations.

Recover

Asks for missing information

If a form asks for data you never supplied, BharatClaw pauses and asks in Telegram instead of inventing an answer.

Product proof

Progress has to be visible while the phone is working.

The floating notch is not decoration. It is the local progress surface: running, waiting for info, continuing, stopped, or done. When overlay permission is unavailable, the foreground notification still carries the task status.

Setup

Install, connect Telegram, enable permissions, run jobs.

1

Create a Telegram bot

Use BotFather, paste the token into BharatClaw, then pair your chat.

2

Add an AI key

Pick Gemini, Claude, or OpenAI inside onboarding and save the key.

3

Enable Accessibility

This is what lets BharatClaw operate the Android UI on your behalf.

4

Grant optional relay/notch permissions

Notification access powers app replies; overlay access powers the notch.
