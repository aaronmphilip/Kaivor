What it is

A phone-side worker, not another chat screen.

The product only matters if it touches the real Android UI. BharatClaw should inspect what is on screen, choose the next action, and ask you when it needs information instead of making up answers.

01

Website and form work

Open a page, read required fields, fill what it knows, and pause in Telegram when a value is missing.

02

App errands

Check prices, delivery status, maps, files, and app results.

03

Replies and notifications

Use Telegram as the control room for supported phone actions.

04

Documents

Read downloads, summarize pages, make PDFs, and send outputs back.

05

Repeatable routines

Run saved macros and repeat jobs that worked before.

The brain

Plan, observe, act, recover.

Skills handle known tasks. The screen agent handles open-ended app navigation. The point is not a pretty chatbot; the point is getting the phone to finish the job.

Plan

Turn your Telegram request into a runnable task.

Observe

Read visible buttons, fields, lists, and warnings.

Act

Tap, type, scroll, open apps, and verify changes.

Recover

Ask for missing info instead of guessing.

Visible progress

The notch shows what the phone is doing.

Running, waiting, continuing, stopped, or done should be visible. If overlay permission is not enabled, the foreground notification still carries the task status.
