What it is

A worker for real Android screens.

The app is useful only if it touches the real UI. BharatClaw should inspect the current screen, choose the next action, and ask you when it needs data instead of guessing.

01

Website and form work

Open a page, read required fields, fill known details, and pause in Telegram when a value is missing.

02

App errands

Check prices, delivery status, maps, files, and app results.

03

Replies and notifications

Use Telegram as the control room for supported phone actions.

04

Documents

Read downloads, summarize pages, make PDFs, and send outputs back.

05

Repeatable routines

Run saved macros and repeat jobs that worked before.

The brain

Plan, observe, act, recover.

Skills handle known tasks. The screen agent handles open-ended app navigation. The goal is not chat. The goal is a finished phone job.

Plan

Turn your Telegram request into a runnable task.

Observe

Read visible buttons, fields, warnings, and app state.

Act

Tap, type, scroll, open apps, and verify changes.

Recover

Ask for missing info instead of guessing.

Visible progress

The phone should show what is happening.

Running, waiting, continuing, stopped, or done should be visible. If overlay permission is off, the foreground notification still carries task status.
